## Codebase Patterns
- RDS proxy resources use `@SDKResource` annotation (SDK V2 pattern), not Framework
- Proxy resources reuse `findDBProxyByName()`, `waitDBProxyUpdated()` from proxy.go
- Composite resource IDs use `/` separator with `SplitN` parsing (see proxy_target.go)
- Auth configs on proxy use `UserAuthConfig` (input) vs `UserAuthConfigInfo` (output) types
- Export test vars in `exports_test.go` alphabetically within sections
- `make gen` must be run after adding `@SDKResource` annotations to register resources
- `validIdentifier` is the standard validator for db_proxy_name fields
- `verify.ValidARN` is used for ARN validation
- Cross-service client access works from any package: `meta.(*conns.AWSClient).IAMClient(ctx)`
- IAM SDK types alias: `awsiam "github.com/aws/aws-sdk-go-v2/service/iam/types"` avoids RDS types conflict
- `tfresource.NewEmptyResultError()` takes no arguments
- IAM policy documents come URL-encoded from AWS; use `url.QueryUnescape()` then `verify.PolicyToSet()`
