# Resource: aws_cloudtrail_insight_selectors

Manages CloudTrail [Insights](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/logging-insights-events-with-cloudtrail.html) event logging for a CloudTrail trail or event data store. Insights analyze management API call rates and error rates to detect unusual activity.

This resource allows Insights configuration to be managed independently from the trail or event data store, making it possible to enable Insights from a separate Terraform module.

~> **Note:** Exactly one of `trail_name` or `event_data_store` must be specified.

~> **Note:** Insights on data events are not supported on event data stores. For event data stores, only management event Insights (`ApiCallRateInsight` and `ApiErrorRateInsight`) are supported.

~> **Note:** To enable `ApiCallRateInsight`, the trail or event data store must log write management events. To enable `ApiErrorRateInsight`, the trail or event data store must log read or write management events.

## Example Usage

### Trail Insights

```terraform
resource "aws_cloudtrail" "example" {
  name                          = "example"
  s3_bucket_name                = aws_s3_bucket.example.id
  include_global_service_events = false
}

resource "aws_cloudtrail_insight_selectors" "example" {
  trail_name = aws_cloudtrail.example.arn

  insight_selector {
    insight_type = "ApiCallRateInsight"
  }

  insight_selector {
    insight_type = "ApiErrorRateInsight"
  }
}
```

### Event Data Store Insights

To enable Insights on an event data store, a source event data store (which logs management events) and a separate destination event data store (which receives Insights events) are required.

```terraform
resource "aws_cloudtrail_event_data_store" "source" {
  name = "example-source"
}

resource "aws_cloudtrail_event_data_store" "destination" {
  name = "example-destination"
}

resource "aws_cloudtrail_insight_selectors" "example" {
  event_data_store     = aws_cloudtrail_event_data_store.source.arn
  insights_destination = aws_cloudtrail_event_data_store.destination.arn

  insight_selector {
    insight_type = "ApiCallRateInsight"
  }
}
```

## Argument Reference

This resource supports the following arguments:

- `region` - (Optional) Region where this resource will be [managed](https://docs.aws.amazon.com/general/latest/gr/rande.html#regional-endpoints). Defaults to the Region set in the [provider configuration](https://registry.terraform.io/providers/hashicorp/aws/latest/docs#aws-configuration-reference).
- `trail_name` - (Optional) Name or ARN of the CloudTrail trail on which to enable Insights. Exactly one of `trail_name` or `event_data_store` must be set. Cannot be used with `event_data_store` or `insights_destination`.
- `event_data_store` - (Optional) ARN of the source event data store on which to enable Insights. Exactly one of `trail_name` or `event_data_store` must be set. Must be used together with `insights_destination`.
- `insights_destination` - (Optional) ARN of the destination event data store that receives Insights events generated by the source `event_data_store`. Required when `event_data_store` is set.
- `insight_selector` - (Required) One or more Insights types to enable. See [`insight_selector`](#insight_selector) below.

### `insight_selector`

`insight_selector` supports the following arguments:

- `insight_type` - (Required) The type of Insights event to log. Valid values are `ApiCallRateInsight` and `ApiErrorRateInsight`.
- `event_categories` - (Optional) List of event categories on which to enable Insights. Valid values are `Management` and `Data`. If omitted, Insights are enabled on management events by default. Specifying `event_categories` overwrites any previously configured categories — for example, if a trail already has Insights enabled on `Management` events and a new request specifies only `Data`, management Insights will be disabled. Insights on data events are not supported on event data stores.

## Attribute Reference

This resource exports the following attributes in addition to the arguments above:

- `id` - ARN of the trail (trail mode) or event data store (event data store mode).

## Import

In Terraform v1.5.0 and later, use an [`import` block](https://developer.hashicorp.com/terraform/language/import) to import CloudTrail Insight Selectors using the trail ARN or event data store ARN. For example:

**Trail mode:**

```terraform
import {
  to = aws_cloudtrail_insight_selectors.example
  id = "arn:aws:cloudtrail:us-east-1:123456789012:trail/example"
}
```

**Event data store mode:**

```terraform
import {
  to = aws_cloudtrail_insight_selectors.example
  id = "arn:aws:cloudtrail:us-east-1:123456789012:eventdatastore/22333815-4414-412c-b155-dd254033gfhf"
}
```

Using `terraform import`, import CloudTrail Insight Selectors using the trail ARN or event data store ARN. For example:

```console
% terraform import aws_cloudtrail_insight_selectors.example arn:aws:cloudtrail:us-east-1:123456789012:trail/example
```
